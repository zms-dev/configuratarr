# Nix DevShell

All development happens inside the Nix devshell defined in `flake.nix`. Enter with:
```
nix develop
```

## What's Available

`defaultShell` `buildInputs` (current):
- Rust toolchain: `rustc`, `cargo`, `clippy`, `rustfmt`, `rust-src` (stable via fenix)
- `python3`, `pkg-config`, `openssl`
- `cargo-nextest`, `shellcheck`, `jq`, `curl`
- OpenAPI tools (on PATH): `list_resources`, `get_resource`, `list_paths`, `get_path`, `compare_schemas`, `diff_resource`, `gen_resource_model`

The OpenAPI tools are shell script wrappers generated by `mkTool`:
```nix
mkTool = name: pkgs.writeShellScriptBin name ''
  exec ${pkgs.python3}/bin/python3 ${toString ./.}/tools/${name}.py "$@"
'';
```

## Adding a New Tool or Utility

**Always add to `flake.nix` `buildInputs` — do not use one-off `nix-shell -p` for anything recurring.**

Pattern for a new nixpkgs package:
```nix
buildInputs = with pkgs; [
  rustToolchain
  python3
  # ... existing ...
  new-package-name   # ← add here
] ++ tools;
```

Pattern for a new Python script wrapper (same as existing OpenAPI tools):
```nix
tools = map mkTool [
  "list_resources"
  # ...
  "new_tool_name"   # ← add here, script must exist at tools/new_tool_name.py
];
```

After editing `flake.nix`, user must re-enter the devshell (`exit` then `nix develop`) to pick up changes.

## E2E Shell

A separate `devShells.e2e` shell adds the actual *arr service binaries (radarr, sonarr, prowlarr, lidarr, readarr):
```
nix develop .#e2e
```
Use only for running end-to-end tests against live service instances.

## Executing Tools from Bash

The Bash tool inherits the user's current shell environment. Python and all project tools exist **only inside the devshell** — never on the host system.

If already inside `nix develop`, all `buildInputs` tools are on PATH:
```bash
get_resource specs/radarr-v3.json "#/components/schemas/DelayProfileResource"
list_resources specs/radarr-v3.json --crud
cargo test -p radarr-v3
```

If the shell is **not** inside `nix develop`, prefix every command:
```bash
nix develop --command get_resource specs/radarr-v3.json "#/components/schemas/DelayProfileResource"
nix develop --command cargo build
```

Check whether devshell is active: `echo $IN_NIX_SHELL` returns `"impure"` inside `nix develop`. If empty, use the `nix develop --command` prefix.

## One-Off Use

`nix-shell -p <pkg>` is acceptable only for truly ephemeral, never-again operations. If you find yourself reaching for it, add to `buildInputs` instead.
